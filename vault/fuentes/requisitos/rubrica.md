# Rúbrica de Evaluación de Informes

Este documento contiene la matriz completa de evaluación para informes técnicos del Departamento de Ingeniería Eléctrica. La ponderación total suma **100 puntos**, distribuidos en cinco dimensiones principales.

## Matriz General de Evaluación

| **Dimensión** | **Ponderación Sección** | **Criterio** | **Indicador de Logro** | **Máx.** | **Logrado** | 
| :--- | :---: | :--- | :--- | :---: | :---: |
| **Aspectos formales** | **20** | Portada | Incluye título, autores/grupo, fecha y logos institucionales, profesor y ayudantes. | **4** |  | 
|  |  | Índices | Presenta índice general e índice de figuras/tablas; si es digital, facilita la navegación. | **4** |  | 
|  |  | Redacción, ortografía y gramática | Lenguaje claro y preciso; sin errores relevantes; redacción en tercera persona/impersonal cuando corresponde. Siempre utilizando la norma IEEE. | **6** |  | 
|  |  | Formato general y numeración | Documento ordenado, limpio, con secciones delimitadas y numeración de páginas visible. | **2** |  | 
|  |  | Esquemas, diagramas, tablas y circuitos | Esquemas claros, prolijos, bien rotulados y, de ser posible, en formato vectorial. Utilizando lo esperado de la norma IEEE para informes técnicos. | **4** |  | 
| **Introducción y fundamento** | **22** | Resumen ejecutivo | Resume objetivos, metodología, resultados clave y conclusiones principales. | **12** |  | 
|  |  | Introducción y objetivos | Contextualiza el problema definiendo objetivos, alcance y limitaciones de manera coherente. Resulta imperativo que se establezca la tesis central de los ensayos subsiguientes, empleando la terminología técnica adecuada. | **5** |  | 
|  |  | Marco teórico y preinforme | Fundamenta con conceptos, modelos, normas y antecedentes relevantes para la experiencia. | **5** |  | 
| **Desarrollo y análisis** | **35** | Desarrollo de la experiencia / metodología | Explica procedimiento, instrumentos, condiciones, extracción de datos y software utilizado. | **10** |  | 
|  |  | Presentación de datos | Tablas, figuras y resultados presentados con títulos, unidades, fuente y lectura clara. | **8** |  | 
|  |  | Presentación de gráficos | Los ejes deben incluir unidades y escalas legibles, junto con leyendas claras y consistencia en las comparaciones. Si no se presentan gráficos en esta sección, el puntaje correspondiente se asignará a la sección anterior. | **4** |  | 
|  |  | Debate y comparación teórico-práctica | Analiza discrepancias, limitaciones, supuestos y conecta teoría con práctica con argumentos. | **13** |  | 
| **Cierre del informe** | **13** | Conclusiones | Debe sintetizar los hallazgos e interpretar los resultados, aportando evidencia empírica que valide la utilidad y aplicabilidad de la teoría propuesta en la práctica. | **13** |  | 
| **Trazabilidad documental** | **10** | Referencias | Referencias completas y consistentes, idealmente bajo norma IEEE o equivalente indicada. | **5** |  | 
|  |  | Anexos | Incluya material complementario útil: fotos, tablas completas, esquemas u otros respaldos. Es obligatorio adjuntar los datos de la experiencia trabajada. | **5** |  | 
| **TOTAL** | **100** |  |  | **100** |  | 




### Resumen de Distribución de Puntajes (Puntos Críticos)

[Aspectos Formales: 20%] ──► [Portada (4pt), Índices (4pt), Redacción/IEEE (6pt), Formato (2pt), Esquemas Vectoriales (4pt)]
[Intro y Fundamento: 22%] ──► [Resumen Ejecutivo (12pt), Introducción/Objetivos (5pt), Marco Teórico (5pt)]
[Desarrollo y Análisis: 35%] ► [Metodología (10pt), Presentación de Datos (8pt), Gráficos (4pt), Debate Teórico-Práctico (13pt)]
[Cierre del Informe: 13%] ──► [Conclusiones basadas en Evidencia (13pt)]
[Trazabilidad Doc.: 10%] ───► [Referencias IEEE (5pt), Anexos con Datos Obligatorios (5pt)]
